# Security Policy / سیاست امنیتی

## Supported Versions / نسخه‌های پشتیبانی‌شده

| Version | Supported |
|---------|-----------|
| 1.0.x   | ✅ Yes    |

## Reporting a Vulnerability / گزارش آسیب‌پذیری

**English:**
If you discover a security vulnerability, please do NOT open a public issue.
Instead, send an email to: ali.pornori@gmail.com

Please include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

**فارسی:**
اگه آسیب‌پذیری امنیتی پیدا کردید، لطفاً Issue عمومی باز نکنید.
به جای آن ایمیل بزنید: ali.pornori@gmail.com

## Security Best Practices / بهترین روش‌های امنیتی

- Never commit API keys or tokens
- Always use `.env` files for sensitive data
- Keep dependencies updated
- Use virtual environments
